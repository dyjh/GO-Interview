## 阅读路线

如果时间有限，建议按面试高频度阅读：

1. **Go 基础能力**：`Go基础`、`类型系统`、`并发编程`、`GMP/GC`。
2. **数据库高频题**：`MySQL基础-事务-锁`、`MySQL索引-执行计划-优化`。
3. **中间件与一致性**：Redis 数据结构、缓存问题、分布式锁、限流、消息队列。
4. **微服务与网络**：REST/RPC、HTTP/TCP、服务发现、ZooKeeper/etcd/Consul。
5. **系统设计题**：秒杀系统、大分页、读写分离、分库分表、缓存和 MQ 串联起来看。
6. **运维补充**：Docker 和 Nginx 用于覆盖部署、反向代理、负载均衡、限流等常见追问。

## 使用建议

- 每篇先看主干知识点，再看“易错点”“注意点”“适用场景”。
- 看到对比表时，重点记边界：什么时候适合、什么时候不适合。
- 看到代码示例时，重点理解它解决的问题，而不是只背语法。
- 最后用每篇的“面试追问”自测，练习把答案说成完整的一段话。
