# Go 基础

## 核心定位

Go 是一门静态类型、编译型、语法简洁、运行时自带调度器和垃圾回收的语言。它适合构建网络服务、微服务、基础设施工具、CLI、云原生组件和高并发 I/O 系统。

面试回答 Go 基础时，重点不要只背语法，而要体现这些特征：

- 编译型语言，部署简单，二进制文件可直接运行。
- 静态类型，类型系统较克制，组合优先于继承。
- 内置 goroutine、channel、select 和 runtime scheduler，天然支持并发。
- 有 GC，开发效率高，但仍要理解逃逸、内存分配和 GC 成本。
- 标准库完整，尤其是 `net/http`、`context`、`encoding/json`、`sync`、`time`、`os` 等。

## 变量、常量和零值

Go 中每个变量都有明确类型和零值：

| 类型 | 零值 |
| --- | --- |
| 数字类型 | `0` |
| bool | `false` |
| string | `""` |
| 指针、slice、map、channel、func、interface | `nil` |
| struct | 每个字段为对应类型零值 |

常见声明方式：

```go
var a int
var name = "go"
count := 10
const timeout = 3
```

注意点：

- `:=` 只能在函数内部使用。
- Go 没有未初始化的局部变量，所有变量都有零值。
- 声明后未使用的局部变量会编译失败，这是 Go 保持代码整洁的方式之一。

## 基本类型

常见基础类型包括：

- 整数：`int`、`int8`、`int16`、`int32`、`int64`、`uint` 等。
- 浮点：`float32`、`float64`。
- 复数：`complex64`、`complex128`。
- 布尔：`bool`。
- 字符串：`string`，不可变字节序列，通常存储 UTF-8 文本。
- 字节和字符：`byte` 是 `uint8` 别名，`rune` 是 `int32` 别名，常用于 Unicode 码点。

面试易错点：

- `len("你好")` 返回字节数，不是字符数。
- 遍历字符串时，`for i, r := range s` 中 `i` 是字节下标，`r` 是 rune。
- Go 不支持隐式数值类型转换，`int32` 和 `int64` 需要显式转换。

## 数组和切片

数组是定长值类型，长度属于类型的一部分：

```go
var a [3]int
b := [3]int{1, 2, 3}
```

切片是更常用的动态序列，内部结构可以理解为：

- 指向底层数组的指针。
- 长度 `len`。
- 容量 `cap`。

```go
s := []int{1, 2, 3}
s = append(s, 4)
```

切片扩容要点：

- `append` 后如果容量足够，会复用原底层数组。
- 如果容量不足，会分配新数组并拷贝旧元素。
- 扩容倍率与 Go 版本、元素大小、运行时策略有关，不能简单背成“容量小于 1024 翻倍，大于 1024 增长 1.25 倍”。面试中更稳妥的说法是：小容量通常增长更快，大容量会逐步降低增长倍率，以平衡拷贝成本和内存浪费。

易错示例：

```go
a := []int{1, 2, 3}
b := a[:2]
b[0] = 100
fmt.Println(a) // [100 2 3]
```

`b` 和 `a` 共享底层数组，所以修改 `b` 会影响 `a`。

## map

`map` 是哈希表，常用于键值存储：

```go
m := make(map[string]int)
m["go"] = 1
v, ok := m["go"]
```

核心要点：

- key 必须是可比较类型，例如 string、数字、指针、数组、结构体中所有字段都可比较时的 struct。
- slice、map、func 不能作为 key。
- 读不存在的 key 返回 value 类型零值，所以要用 `v, ok := m[k]` 判断是否存在。
- map 不是并发安全的，多个 goroutine 同时读写会触发数据竞争，严重时直接 panic。
- map 遍历顺序不保证稳定，不要依赖遍历顺序做业务逻辑、签名或测试断言。

面试踩坑：

- `var m map[string]int` 得到的是 nil map，可以读，读不到返回零值，但不能写，写会 panic；写入前要 `make`。
- map 并发读写要用锁、`sync.Map` 或其他并发结构保护。

实现原理面试版：

- Go map 底层是哈希表，由多个 bucket 组成。
- 每个 bucket 可放多个 key/value，并保存 hash 的高位信息用于快速比较。
- 冲突较多时会使用 overflow bucket。
- 数据增长或冲突过多时会触发扩容，并在后续操作中渐进式迁移旧 bucket。

## struct 和方法

结构体用于表达业务对象：

```go
type User struct {
    ID   int64
    Name string
}
```

方法通过接收者绑定到类型：

```go
func (u User) DisplayName() string {
    return u.Name
}

func (u *User) Rename(name string) {
    u.Name = name
}
```

值接收者和指针接收者区别：

- 值接收者会复制一份接收者，适合小对象和不修改内部状态的方法。
- 指针接收者避免复制，且可以修改原对象。
- 如果类型中包含锁、较大字段、slice/map 等引用语义字段，通常优先用指针接收者。

## 函数

Go 函数支持多返回值：

```go
func div(a, b int) (int, error) {
    if b == 0 {
        return 0, errors.New("divide by zero")
    }
    return a / b, nil
}
```

面试重点：

- 参数默认值传递。传指针、slice、map、channel 时，复制的是描述符或指针值，底层数据仍可能共享。
- 支持匿名函数和闭包。
- 函数可以作为参数和返回值。

闭包常见坑：

```go
for i := 0; i < 3; i++ {
    i := i
    go func() {
        fmt.Println(i)
    }()
}
```

在并发场景里，要避免闭包捕获循环变量造成非预期结果。较新的 Go 版本已经改善了 `range` 循环变量捕获问题，但面试时仍建议显式传参或局部复制，表达意图更清楚。

## defer

`defer` 会在函数返回前执行，多个 defer 按后进先出顺序执行：

```go
func readFile(path string) error {
    f, err := os.Open(path)
    if err != nil {
        return err
    }
    defer f.Close()
    return nil
}
```

注意点：

- defer 的参数会在注册时求值。
- 如果函数有命名返回值，`return` 会先给返回值赋值，再执行 defer，defer 可以修改命名返回值。
- defer 常用于释放资源、解锁、关闭文件、recover panic。
- 在热点循环里大量 defer 可能增加开销，应根据场景权衡。

```go
func f() (n int) {
    defer func() { n++ }()
    return 1 // 最终返回 2
}
```

## 标准库高频点

### `net/http`

```go
http.HandleFunc("/ping", func(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintln(w, "pong")
})
http.ListenAndServe(":8080", nil)
```

追问点：

- `http.Server` 如何设置超时。
- 如何从 request 中获取 context。
- handler 中为什么不能忽略 body 关闭、超时和错误处理。

### `encoding/json`

```go
type User struct {
    Name string `json:"name"`
    Age  int    `json:"age"`
}
```

注意点：

- 只有导出字段才能被 JSON 编码/解码。
- `omitempty` 会省略零值字段。
- 金额、ID 等精度敏感字段不要随意用 `float64`。

### `time`

- `time.Now()` 获取当前时间。
- `time.After` 可用于简单超时，但频繁创建要注意定时器成本。
- `time.NewTicker` 用于周期任务，使用完要 `Stop()`。
- 时间比较用 `Before`、`After`、`Equal`。

## 面试追问

### Go 和 Java/PHP/Python 相比有什么特点？

Go 的优势是部署简单、并发模型轻量、标准库强、工程约束清晰，适合网络服务和云原生场景。缺点是泛型能力相对克制，语言表达力不如一些动态语言灵活，GC 场景下极端低延迟控制不如手动内存管理语言。

### new 和 make 的区别？

`new(T)` 分配 T 类型零值并返回 `*T`。`make` 只用于 slice、map、channel，返回初始化后的对象本身。

#### 参数说明

```go
p := new(int)              // 返回 *int，指向 int 零值 0
s := make([]int, 0, 4)     // slice: len=0, cap=4
m := make(map[string]int, 100) // map: 100 是容量提示，len 仍为 0
ch := make(chan int, 10)   // channel: 缓冲区容量为 10
```

slice 要特别注意：`make([]T, len)` 的第二个参数是长度，元素已经存在；如果后面要用 `append`，通常写 `make([]T, 0, cap)`。

### nil slice 和空 slice 的区别？

```go
var a []int        // nil slice
b := []int{}       // empty slice
```

两者 `len` 都是 0，都可以 `append`。区别在于 `a == nil` 为 true，JSON 编码时 nil slice 通常是 `null`，空 slice 通常是 `[]`。
